# DESIGN.md — もみかるFC募集LP デザインシステム

本ドキュメントは、もみかるフランチャイズ募集LPの全ビジュアル仕様を定義する。実装前に必ず全セクションを通読し、記載された数値をそのまま使用すること。曖昧な形容詞による判断は禁止する。

---

## 1. Visual Theme

- **テーマ名**: Momikaru Signature Yellow
- **コアコンセプト**: ブランド一貫性・信頼感・収益性
- **対象読者**: 30〜50代の事業家・独立志望サラリーマン
- **キーワード**: 歴史12年、100店舗規模、もみかるブランド、投資回収可能性
- **ムード**: 明るく視認性の高いブランドスタイル、事業の健全性を示す前向きな設計
- **禁止ムード**: 情報商材LP的な派手さ、パステル、手書き風、カジュアルSNS風
- **基調**: ブランドイエロー（#F7D257）を全体の70%、オフホワイトを25%、ゴールドを5%の比率で配色

## 7. Do's and Don'ts

### Do
- 数値は必ず確定した値（直営15店舗、FC約85店舗、合計約100店舗、創業12年）で記載する
- CTAボタンはヒーロー・強み直後・収益モデル直後・FAQ直後・フォーム直前の5箇所に配置する
- 見出しは体言止めまたは断定文で記述する
- Brand 500 を画面全体の 70%、Off White を 25%、Gold を 5% の比率で配色する
- 黄色背景上のテキストは必ず Ink 900 (`#0D1117`) を使用する
- 画像にはalt属性を必ず付与する
- 本文テキストは16px以上を維持する

### Don't
- 「豊富な実績」「多数の店舗」など数値を伴わない曖昧な形容詞を使わない
- 絵文字を使わない
- 斜体・手書き風フォント・グラデーション文字を使わない
- ネオン色・パステル色・原色の赤/青/緑を使わない
- 角丸を16px以上にしない（堅実感を損なうため）
- 黄色背景上に白文字を載せない（Pure White on Brand 500 は 1.9:1 でAA未達）
- 黄色背景上にGold 500を重ねない（1.8:1 でAA未達、Gold は Off White / Pure White / Ink 900 上のみ）
- 背景画像の上に直接テキストを載せない（必ず Ink 900 のオーバーレイを 60% 以上で敷く）
- コントラスト比4.5:1未満の配色を使わない
- 収益保証・必ず儲かる等の断定的収益表現を使わない

## 8. Responsive Behavior（モバイルファースト完全版）

本セクションは本LPのレスポンシブ設計の正典である。SP表示の品質がPC表示より優先される点に注意すること。

### 8.1 設計思想の根本原則

- 本LPは **モバイルファースト設計** を採用する
- 標準基準画面は **375px（iPhone標準）**
- 全コンポーネントをまず375pxで最適化し、PC向けに拡張する
- **「PCで設計 → SPに縮小」の順序は絶対禁止**
- **SP表示の品質がPC表示より優先される**
- CSS実装は `min-width` メディアクエリ（モバイル基本、PC拡張）を基本とする

### 8.2 ブレークポイント4段階

| 区分 | 範囲 | 主要端末 |
|------|------|---------|
| **SP標準** | 375px以下 | iPhone SE / 標準、設計起点 |
| **SP大型** | 376px〜767px | iPhone Pro Max、Android大型 |
| **タブレット** | 768px〜1023px | iPad縦、タブレット一般 |
| **PC** | 1024px以上 | ノートPC・デスクトップ |

### 8.3 コンテナ幅

| SP（〜767px） | 20px | 100% |
| タブレット（768〜1023px） | 32px | 100% |
| PC（1024px以上） | 40px | 1200px（中央寄せ） |

### 8.4 タイポグラフィのレスポンシブ

### 8.5 数値表示の絶対ルール（最重要）

§4.5 の数値ハイライト仕様を本ルールで上書きする。数値表示はLP全体で最も誤崩壊しやすい箇所のため、以下を絶対遵守する。

#### 絶対遵守ルール

1. 数値と単位は **常に同一flex行** に配置する
2. 数値と単位を囲む親要素に `white-space: nowrap` を必須適用する
3. 数値と単位は `align-items: baseline` でベースライン揃え
4. 親要素は `flex-wrap: nowrap` を明示する
5. **3カラム数値表示はSPで必ず縦積み（1カラム化）**

#### サイズ仕様

| 区分 | メイン数値 | 単位 | 比率 |
|------|----------|-----|------|
| SP | **48〜64px**（`clamp(48px, 14vw, 64px)`） | 16〜22px | **3:1** |
| PC | **120〜180px** | 32〜48px | **4:1** |

#### 各カラム間の余白

- SP: 48px
- PC: 48px

### 8.6 セクション上下余白

| 区分 | 上下 padding |
|------|-------------|

| タブレット | 96px |

- **固定CTAバー**
  - **SP標準装備**: 画面下部 `position: fixed`、高さ **72px**、ヒーロー離脱（200〜400pxスクロール）後に表示
  - **PC**: 非表示（ヒーローCTAが常に近いため不要）

### 8.9 ナビゲーション

| 区分 | 仕様 |
|------|-----|
| SP | ハンバーガーメニュー、ロゴ左寄せ、高さ 56px |
| PC | 横並びメニュー、ロゴ左寄せ、CTA右寄せ、高さ 64px |

- スクロール後（共通）: 背景に `backdrop-filter: blur(20px)` を適用（§10.7 準拠）
- ハンバーガー展開時は全画面オーバーレイ、リンクは縦並びで各 56px 高さ確保

### 8.10 画像・ビジュアル

- `max-width: 100%`、`height: auto` を全画像に適用
- `aspect-ratio` プロパティで縦横比を明示
- SP時は複雑な図解を簡略化する
  - マトリクス図: **SP時は縦積みリスト化**
  - フローチャート: **SP時は上から下への単純フロー**
  - 比較図表: **SP時はカード型縦積み**

### 8.11 テーブル・比較表（3ステップアッププラン等）

| 区分 | 表示形式 |
|------|---------|
| SP | **カード型縦積み**、各プランを独立カードで全表示 |
| タブレット | 横スクロール対応テーブル |
| PC | 通常の比較テーブル |

- **プラン比較はSPでスワイプ切替を使用しない**。縦積み全表示を必須とする（比較検討層に全情報を一度に提示するため）

### 8.12 タップ領域

- **最小 44×44px**（Apple Human Interface Guidelines 推奨）
- 隣接するタップ要素間に **最低 8px 以上** の間隔を確保
- 誤タップ防止のため、リンクとリンクの間に非タップ可能な空白領域を設ける

### 8.13 SP専用の禁止事項

SP表示における以下の挙動は **全面禁止** とする。

1. 横スクロール発生の **完全禁止**（`overflow-x: hidden` は `<body>` レベルで適用せず、個別要素の width 制御で回避）
2. フォント **12px以下** の使用禁止（可読性確保）
3. ホバーエフェクトに依存した情報表示の禁止（タッチ環境では発火しないため）
4. **SP時の数値・単位の縦積み禁止**（§8.5 絶対ルール違反）
5. 横長テーブルのはみ出し（`overflow-x: auto` でラップする場合も、親コンテナ内に収めること）

### 8.14 アニメーション調整（SP最適化）

§10 のアニメーション仕様を SP 端末で以下のように軽減する。

| 項目 | PC | SP |
|------|----|----|
| translateY 移動距離 | 60px | **30px**（50%短縮） |
| 同時並行アニメーション数 | 最大5個 | **最大3個** |
| パララックス効果 | 有効 | **無効化** |
| Ken Burns（背景拡大） | 10秒 | 無効化またはキーフレーム簡略化 |
| 文字ばらしfade-in stagger | 0.05s | 0.04s（わずかに短縮） |

### 8.15 実装後チェックリスト（SP基準）

- [ ] 375px幅で全セクションを上から下まで確認した
- [ ] 横スクロールが発生していないか検証した（`overflow-x` デバッガー確認）
- [ ] すべての数値＋単位ペアが nowrap で1行に収まっているか
- [ ] 3カラム数値表示がSPで縦積みになっているか
- [ ] 3ステップアッププラン比較がSPで縦積みカード表示になっているか
- [ ] 本文が15px、キャプションが13px以上を維持しているか
- [ ] すべてのタップ領域が44×44px以上あるか
- [ ] ヒーローCTAから可視範囲にいない状態で固定CTAバーが表示されているか
- [ ] ホバー依存の情報がSPでタッチまたは代替UIで到達可能か
- [ ] アニメーション距離が30px、同時可動3個以下に抑えられているか

## 9. Agent Prompt Guide

実装を担当するエージェント（Claude）は、以下の指針に従ってコード生成を行うこと。

### 実装前チェックリスト

- 本DESIGN.mdに定義されていないカラー・サイズを独自に追加しない
- `tailwind.config` 相当のカスタム設定は `<script>` 内の `tailwind.config = {...}` で定義する
- HEXコードは本DESIGN.mdの値をそのまま使用する（近似値に丸めない）
- セマンティックHTML要素（`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`）を使用する

### 禁止プロンプト例
- 「適当にいい感じの黄色で」→ 必ず `#F7D257` / `#FDA92A` / `#FDBA55` から選択

- 「たくさんの実績をアピール」→ 必ず「直営15店舗・FC約85店舗・合計約100店舗・創業12年」の数値で表現

## 10. Animation & Interaction

本セクションはLP全体のアニメーション・インタラクション仕様を定義する。目指す体験は Apple / Tesla / Nike コーポレートサイトに相当する映画的な重厚さと記憶に残る演出である。情報商材LP的な煽り演出・点滅・回転・過剰装飾は全て排除する。記載された数値はそのまま実装値として使用すること。

### 10.1 基本原則

- 動きは「重厚」「滑らか」「映画的」を数値で体現する
- 標準持続時間: **0.8秒**（基本）／ **1.2秒**（ヒーローなど主要演出）
- 標準イージング: `cubic-bezier(0.22, 1, 0.36, 1)`（ease-out-quint、映画的減速）
- `prefers-reduced-motion: reduce` のユーザーに対しては、全アニメーションを無効化する
- 同時に動く要素は **最大5つまで**。5つを超える場合は stagger で時間差を作る

### 10.2 ヒーローセクション演出

| 要素 | 挙動 | 開始タイミング | 持続時間 |
|------|------|--------------|---------|
| メインキャッチコピー | 1文字ずつ fade-in（`opacity: 0 → 1`） | ロード完了後、各文字に **0.05秒** ずつディレイ加算 | 各文字 0.4秒 |
| サブコピー | fade-in + `translateY(40px → 0)` | メインキャッチ完了の **0.4秒後** | 0.8秒 |
| CTAボタン | `scale(0.9 → 1)` + `opacity(0 → 1)` | サブコピー完了の **0.8秒後** | 0.8秒 |
| 背景画像（存在する場合） | Ken Burns エフェクト（`scale(1 → 1.05)`） | ロード完了直後 | **10秒** かけて実行 |
| スクロールインジケーター | 画面下部に配置、上下に **1.5秒周期** でフロート（`translateY(0 → 8px → 0)`） | ロード完了直後 | 1.5秒 / 1サイクル |

### 10.3 スクロールアニメーション

| 項目 | 値 |
|------|---|
| 初期状態 | `opacity: 0; transform: translateY(60px);` |
| 完了状態 | `opacity: 1; transform: translateY(0);` |
| 発火タイミング | 要素の上端が画面下端から **150px** 内側に入った時点 |
| 持続時間 | **0.8秒** |
| イージング | `cubic-bezier(0.22, 1, 0.36, 1)` |
| stagger | 同一セクション内の複数要素は **0.15秒** ずつディレイを加算する |

### 10.4 数値表示の演出

- 対象: 店舗数・年数・金額・比率などの数値表示要素（12、136、450、1500、24、5 等）
- 初期値: `0`
- 終了値: 実数値
- 持続時間: **2.0秒**
- イージング: `cubic-bezier(0.22, 1, 0.36, 1)`
- 発火タイミング: 10.3と同じ（画面下端から 150px 内側進入時）
- 文字サイズ: 見出しの2倍以上、**120px〜200px** の範囲で設定する
- 周囲余白: 数値要素の上下左右に最低 **48px** の余白を確保し、画面の主役として成立させる
- レイアウトシフト防止: タブラー数字（`font-feature-settings: "tnum", "lnum"`）を必須適用する

### 10.5 セクション切り替えの大胆な演出

- 背景色はセクション単位で **ブランドカラー（明）↔ Ink 900（暗）** の交互配置を採用する
  - 推奨順: Brand 500（ブランド）→ Off White（明）→ Ink 900（暗・劇的）→ Brand 500（ブランド）→ Pure White（明）→ Ink 900（暗）の変化
  - 黄色背景セクションは Ink 900 の文字、Ink 900 背景セクションは Brand 500 または Pure White の文字で統一する
- セクション境界では背景色が **画面全体で切り替わる演出** を採用する（ページを縦スクロールした際に背景のコントラストが明確に変わる）
- 重要セクション（**オーナー実績・投資回収シミュレーション**）は最低高さ **100vh** を確保する

### 10.6 ホバーエフェクト（リッチ版）

| 対象 | 変化 | 持続時間 |
|------|------|---------|
| カード | `translateY(0 → -8px)` + box-shadow を e1 → e3 へ拡大 + 背景に微細なグラデーション出現 | 0.4秒 |
| ボタン | 背景に光が左から右へ走る shine エフェクト（`linear-gradient(90deg, transparent, rgba(255,255,255,0.25), transparent)` を `translateX(-100% → 100%)`） | 0.6秒 |
| リンク | 下線が `width: 0 → 100%` で左から右へ伸長 + 文字色が Brand 700 へ変化 | 0.4秒 |
| 画像 | `scale(1 → 1.08)` + `filter: brightness(1 → 1.05)`、親要素に `overflow: hidden` 必須 | 0.6秒 |

- カード box-shadow 値は §6 Depth & Elevation の定義に従う（e1 → e3）
  - e1: `0 1px 2px rgba(13, 17, 23, 0.06)`
  - e3: `0 16px 48px rgba(13, 17, 23, 0.18)`

### 10.7 マイクロインタラクション

| 対象 | 挙動 | 持続時間 |
|------|------|---------|
| フォーム入力フィールド | フォーカス時に枠線色を Ink 900 (`#0D1117`) へ変化 + `box-shadow: 0 0 0 4px rgba(253, 169, 42, 0.24)` の Brand 700 グロー | 0.3秒 |
| アコーディオン（FAQ） | `max-height` スムーズ展開 + チェブロンアイコン `rotate(0 → 180deg)` | 0.5秒 |
| ナビゲーション | スクロール位置が **80px** を超えた時点で、ヘッダー背景を `backdrop-filter: blur(20px)` + `background-color: rgba(247, 210, 87, 0.82)` へ変化（Brand 500ベース） | 0.3秒 |

### 10.8 グリッドからのはみ出し演出

- ヒーロー画像は意図的にコンテナグリッド（max-width: 1200px）を超え、画面右端まで到達させる配置を採用可能
- セクション境界では要素を跨がせる配置を採用可能（例: 前セクション末尾のカードを次セクションに **−80px** の負マージンで食い込ませる）
- 大きな数字・見出しは画面左端から **0px 〜 40px** 以内に到達する左寄せを活用する

### 10.9 禁止事項

- ループする無限アニメーションは全面禁止（**点滅・回転・マーキー・カルーセル自動再生を含む**）
- 文字の回転・歪み・3D回転（`rotate`・`skew`・`rotateX/Y/Z`）は全面禁止
- 「今だけ」「残り◯名」「締切まで◯秒」系の煽りバナー・カウントダウンタイマーは全面禁止
- 派手すぎるグラデーション（**3色以上** の色を混ぜた背景）は全面禁止
- 動画背景は使用可だが、再生速度は **0.5倍速** まで落とすこと（`playbackRate = 0.5`）
- 効果音・音声の自動再生は絶対禁止（`autoplay` + 音声トラック付きメディア禁止）

### 10.10 実装方法

- スクロール進入検知は **IntersectionObserver API** を使用する
- CSS の `transition` / `transform` / `opacity` / `filter` を優先使用し、GPU合成を活用する（`top` / `left` / `margin` / `width` / `height` のアニメーションは避ける）
- 複雑なタイムラインが必要な演出は **GSAP**（CDN経由）を使用する
- スクロール連動演出は **ScrollTrigger**（GSAP公式プラグイン、CDN経由）の使用を許可する
- **jQuery の使用を禁止する**
- reduced-motion 対応として以下を必ず実装する:

  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      animation-iteration-count: 1 !important;
      transition-duration: 0.01ms !important;
      scroll-behavior: auto !important;
    }
  }
  ```

### 10.11 パフォーマンス制約

- アニメーション中の FPS は **60** を維持する（Chrome DevTools の Performance タブで確認）
- 同時実行アニメーション数は画面内 **最大5個**
- `filter: blur()` の **常時適用を禁止**（ヘッダーの backdrop-filter: blur(20px) のような局所適用のみ許可）
- `will-change` プロパティは適用直前に JavaScript で動的設定し、アニメーション終了後は `will-change: auto` に解除する
- 画像は `loading="lazy"` を適用し、初期ビューポート外の画像はスクロール接近時に読み込む
